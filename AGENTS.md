# AGENTS.md

本仓库的 AI 代理必须按以下顺序工作，禁止跳步。

## 一、执行顺序
1. 先阅读 `HOOK.md`
2. 完成项目分类、风险分级、仓库探测
3. 创建或更新任务日志
4. 再阅读 `STANDARDS.md`
5. 若任务涉及上线、灰度、回滚、线上故障，额外执行对应运行手册（如仓库存在 `RUNBOOK.md`）
6. 完成方案、测试计划后，才能进入实现

## 二、文档分工
- `HOOK.md`：任务开始时的初始化动作
- `STANDARDS.md`：方案、测试、实现、文档更新、交付结论
- `RUNBOOK.md`：线上故障、发布、回滚、灰度（如存在）

## 三、任务类型与必读路径
- 全新项目：`HOOK.md` → `STANDARDS.md`
- 新增功能：`HOOK.md` → `STANDARDS.md`
- 重构优化：`HOOK.md` → `STANDARDS.md`
- 性能优化：`HOOK.md` → `STANDARDS.md`
- 线上问题：`HOOK.md` → `RUNBOOK.md` → `STANDARDS.md`

## 四、硬性规则
- 禁止跳过项目分类
- 禁止跳过风险分级
- 禁止跳过仓库探测
- 禁止没有测试计划就直接实现
- 禁止高风险变更没有回滚说明
- 禁止代码改动后不更新文档和任务日志

## 五、默认目标优先级
1. 正确性
2. 兼容性
3. 可维护性
4. 可测试性
5. 可观测性
6. 安全性
7. 可回滚性
8. 性能与成本平衡

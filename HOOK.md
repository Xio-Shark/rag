# HOOK.md

收到任何任务后，禁止直接编码。必须先完成以下步骤。

## 一、项目分类
先判断任务主类型：

- A. 全新项目
- B. 现有项目新增功能
- C. 成熟项目优化 / 重构
- D. 线上问题修复
- E. 性能 / 成本专项
- F. 原型 / Demo

若同时符合多个类型，选一个主类型，并注明次类型。

## 二、风险分级
- R0：文档或无逻辑变化
- R1：低风险局部改动
- R2：关键模块或关键路径改动
- R3：数据库、公共接口、认证、部署、生产行为改动

不确定时，默认升一级。

## 三、仓库探测
检查并记录以下内容是否存在：

- README
- 架构文档
- 实施计划
- 测试框架
- CI
- 环境变量模板
- 日志机制
- 迁移机制
- 特性开关
- 回滚方式

输出：
- 已存在项
- 缺失项
- 本轮必须先补齐的项

## 四、初始化动作

### A. 全新项目
先补：
- 目录骨架
- README
- architecture 文档
- implementation-plan 文档
- `.env.example`
- 测试骨架
- 最小 CI
- 日志约定

### B. 新增功能
先做：
- 识别受影响模块
- 标记兼容性要求
- 标记回归测试范围
- 判断是否需要特性开关

### C. 重构优化
先做：
- 明确允许变化与禁止变化
- 建立行为基线
- 判断是否需要灰度、开关、ADR

### D. 线上问题
先做：
- 记录现象与影响范围
- 区分止血与根因修复
- 保留证据
- 优先最小修复

### E. 性能 / 成本
先做：
- 建立基线
- 确认瓶颈
- 设计前后对比方法

### F. 原型 / Demo
先做：
- 明确边界
- 标明哪些实现不可直接进生产
- 列出产品化缺口

## 五、任务日志
每轮任务都要创建或更新：

`docs/agent-runs/YYYY-MM-DD-任务名.md`

至少记录：
- 任务名称
- 项目类型
- 风险等级
- 目标
- 约束
- 方案摘要
- 关键决策
- 变更文件
- 风险与未覆盖项
- 回滚方式

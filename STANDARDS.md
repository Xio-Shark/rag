# STANDARDS.md

完成 HOOK 后，按以下规则进入正式执行阶段。

## 一、正式执行前必须输出
1. 目标
2. 非目标
3. 约束
4. 风险
5. 验收标准
6. 当前项目类型
7. 当前风险等级
8. 仓库现状
9. 缺失机制清单
10. 方案对比
11. 推荐方案
12. 是否需要 ADR
13. 测试计划
14. 实施步骤

## 二、方案规则
至少给出 2 个方案，对比：
- 复杂度
- 性能
- 成本
- 侵入性
- 兼容性
- 维护成本
- 回滚难度

## 三、ADR 触发条件
出现以下情况必须写 ADR：
- 新增基础设施依赖
- 修改数据库结构
- 修改认证授权
- 修改核心调用链
- 引入缓存 / 队列 / 异步任务
- 大范围重构
- 高风险接口变更

## 四、测试优先
先写测试，再写实现。至少考虑：
- 验收测试
- 回归测试
- 边界测试
- 集成测试
- 性能 / 安全测试（按需）

## 五、实现要求
- 优先增量修改
- 优先复用成熟能力
- 优先小模块、清晰边界
- 关键逻辑写中文注释
- 禁止硬编码敏感信息

## 六、必查项
- 兼容性
- 日志 / 指标 / trace
- 输入校验与权限校验
- 特性开关 / 灰度 / 回滚
- 文档更新
- 任务日志更新

## 七、交付结论必须包含
- 风险清单
- 未覆盖项
- 上线检查项
- 回滚步骤
- 后续建议
